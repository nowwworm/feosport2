1Обзор системы

FeoSport2 — веб-приложение для управления соревнованиями по FPV-дронрейсингу. Работает в локальной сети без подключения к интернету. Судьи вносят результаты со смартфонов, табло обновляется в реальном времени у всех зрителей.

КомпонентТехнологияНазначение
BackendNode.js 20 + Express + Socket.ioREST API, WebSocket, бизнес-логика
База данныхPostgreSQL 16Хранение пилотов, заездов, результатов
FrontendReact 18 + Vite + SCSSWeb-интерфейс (PWA, мобильный)
ПроксиNginx (Docker) / прямой (нативный)Единая точка входа

Три варианта установки на Windows 11:

3Вариант А — Установка через exe-инсталлятор

Это рекомендуемый способ для развёртывания на пользовательских машинах. Инсталлятор содержит предсобранный сервер, фронтенд и автоматически настраивает базу данных.

3.1 Сборка инсталлятора (на машине разработчика)

Сборка выполняется один раз на любой Windows-машине с установленным Git и доступом в интернет. Результат — файл FeoSport2-Setup.exe, который затем копируется на целевые машины.
  1. 1
    Скопировать исходники проекта Перенести папку feosport2 на Windows-машину (USB, сеть, архив). Полный путь, например: C:\feosport2\
  2. 2
    Открыть PowerShell от Администратора Нажать Win, ввести «PowerShell», правая кнопка → «Запуск от имени администратора».
  3. 3
    Разрешить выполнение скриптов 
    Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
  4. 4
    Перейти в папку проекта и запустить сборку 
    cd C:\feosport2
.\deploy\windows\build\build-installer.ps1
    Скрипт выполнит следующее автоматически:
    • Установит зависимости npm для backend и frontend
    • Скомпилирует Node.js сервер в feosport2-server.exe через pkg
    • Соберёт React-приложение через Vite
    • Скачает PostgreSQL 16 installer (~300 МБ)
    • Установит Inno Setup 6 через winget
    • Скомпилирует финальный инсталлятор
  5. 5
    Результат По завершении файл будет здесь: 
    C:\feosport2\deploy\windows\output\FeoSport2-Setup.exe
    Размер приблизительно 350–400 МБ (включает PostgreSQL installer).
Если интернет недоступен во время сборки, заранее скачай PostgreSQL 16 вручную с сайта postgresql.org → Downloads → Windows и положи файл в deploy\windows\build\deps\postgresql-16-win-x64.exe

3.2 Установка на целевой машине

  1. 1
    Скопировать FeoSport2-Setup.exe на целевую машину USB-накопитель, локальная сеть или любой удобный способ.
  2. 2
    Запустить инсталлятор от имени Администратора Правая кнопка на файле → «Запуск от имени администратора».
    Если появится SmartScreen — нажать «Подробнее» → «Выполнить в любом случае».
  3. 3
    Экран выбора компонентов
    КомпонентКогда выбирать
    Приложение FeoSport2Всегда (обязательный)
    PostgreSQL 16Если PostgreSQL ещё не установлен
  4. 4
    Ввести пароль postgres (суперпользователь) Если PostgreSQL устанавливается впервые — придумать новый пароль.
    Если уже установлен — ввести существующий пароль.
    Запомните этот пароль!
  5. 5
    Ввести пароль пользователя feosport Это технический пароль, который приложение использует для подключения к БД.
    Можно оставить значение по умолчанию (feosport2024) или задать своё.
  6. 6
    Дождаться завершения установки Установщик автоматически:
    • Установит PostgreSQL 16 (если выбран)
    • Создаст базу данных feosport2
    • Применит схему и загрузит тестовые данные
    • Создаст ярлык на рабочем столе
  7. 7
    Выбрать задачи при завершении
    • ☑ Создать ярлык на рабочем столе — рекомендуется
    • ☑ Запускать автоматически при входе — для постоянного сервера
    • ☑ Запустить FeoSport2 сейчас — откроет браузер сразу

После установки дважды кликнуть ярлык FeoSport2 на рабочем столе.

В браузере откроется страница входа по адресу http://localhost:8090

4Вариант Б — Docker Desktop

Этот вариант запускает все компоненты в изолированных контейнерах. Идеален для тестирования или когда нужна воспроизводимая среда.

4.1 Установка Docker Desktop

  1. 1
    Открыть PowerShell от Администратора и запустить скрипт установки 
    Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
cd C:\feosport2
.\deploy\windows\docker\01-install-docker.ps1
    Скрипт включит WSL2, установит Docker Desktop через winget и добавит пользователя в группу docker-users.
  2. 2
    Перезагрузить компьютер Скрипт предложит перезагрузку автоматически. Нажать Y или перезагрузить вручную.
  3. 3
    Дождаться запуска Docker Desktop После перезагрузки Docker Desktop запустится автоматически. В системном трее (правый нижний угол) появится иконка кита. Подождать, пока надпись сменится с «Starting» на «Docker Desktop is running».
На Windows 11 Home Docker Desktop требует WSL2. Если появляется ошибка виртуализации — проверить, включена ли виртуализация в BIOS/UEFI (раздел CPU Configuration → Intel Virtualization Technology = Enabled).

4.2 Настройка и запуск

  1. 1
    Настроить переменные окружения 
    .\deploy\windows\docker\02-setup-env.ps1
    Скрипт создаст файл deploy\.env с паролями и JWT-секретом. Откроет диалог для ввода пароля PostgreSQL.
  2. 2
    Запустить приложение 
    .\deploy\windows\docker\start.bat
    Docker скачает образы и соберёт контейнеры (первый запуск ~5–10 минут).
  3. 3
    Загрузить тестовые данные 
    .\deploy\windows\docker\seed.bat
  4. 4
    Открыть браузер Сайт доступен по адресу: http://localhost:80 (или просто http://localhost)
ФайлДействие
start.batЗапустить все контейнеры
stop.batОстановить (данные сохраняются)
logs.batПросмотр логов всех сервисов
seed.batЗагрузить тестовые данные

5Вариант В — Нативная установка

Этот вариант не использует Docker. Node.js и PostgreSQL устанавливаются непосредственно в Windows.

5.1 Установка Node.js и PostgreSQL

  1. 1
    Открыть PowerShell от Администратора 
    Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
cd C:\feosport2
.\deploy\windows\native\01-install-deps.ps1
    Скрипт через winget установит:
    • Node.js 20 LTS (OpenJS.NodeJS.LTS)
    • PostgreSQL 16 (PostgreSQL.PostgreSQL.16)
    И выполнит npm install для backend и frontend.
  2. 2
    Если winget недоступен — ручная установка
    ПОСсылка для скачивания
    Node.js 20 LTShttps://nodejs.org/en/download → Windows Installer (.msi)
    PostgreSQL 16https://www.postgresql.org/download/windows/ → Download the installer
    После ручной установки выполнить: 
    cd C:\feosport2\backend  &&  npm install
cd C:\feosport2\frontend &&  npm install
После установки закрыть и снова открыть PowerShell, чтобы переменные PATH обновились.

5.2 Настройка базы данных

  1. 1
    Запустить скрипт настройки БД 
    .\deploy\windows\native\02-setup-db.ps1
    Скрипт создаст пользователя feosport, базу данных feosport2, применит схему и запишет backend\.env.
  2. 2
    Загрузить тестовые данные 
    .\deploy\windows\native\seed.bat

5.3 Запуск приложения

.\deploy\windows\native\start.bat

Откроются два окна: FeoSport2 Backend (порт 8090) и FeoSport2 Frontend (порт 8080). Сайт доступен по адресу: http://localhost:8080

Не закрывать окна командной строки — это остановит сервер. Для остановки использовать stop.bat.

7.3 Доступ с других устройств (смартфоны судей)

Для работы в локальной сети судьи открывают браузер на телефоне и вводят IP-адрес компьютера-сервера.

  1. 1
    Узнать IP-адрес сервера 
    ipconfig
    Найти строку «IPv4-адрес» в разделе своего сетевого адаптера.
    Например: 192.168.1.100
  2. 2
    На телефоне открыть браузер и ввести адрес 
    http://192.168.1.100:8090   (Вариант А)
http://192.168.1.100        (Вариант Б)
http://192.168.1.100:8080   (Вариант В)
Приложение является PWA — судьи могут добавить его на главный экран телефона через меню браузера «Добавить на главный экран» для удобного доступа.

9Устранение неисправностей

9.1 Сервер не запускается

СимптомПричинаРешение
Браузер показывает «Страница недоступна» Сервер не запущен Запустить start-feosport.bat
Ошибка «Port 8090 already in use» Порт занят другой программой netstat -ano | findstr 8090, затем taskkill /PID <PID> /F
Ошибка подключения к БД PostgreSQL не запущен net start postgresql-x64-16
«password authentication failed» Неверный пароль в .env Проверить C:\Program Files\FeoSport2\.env

9.2 Проблемы с Docker Desktop

СимптомРешение
«Hyper-V not enabled» Включить виртуализацию в BIOS и выполнить: bcdedit /set hypervisorlaunchtype auto
Docker не запускается после перезагрузки Запустить Docker Desktop вручную из меню Пуск
Контейнер backend не стартует Проверить логи: deploy\windows\docker\logs.bat

9.3 Сброс данных

Следующие команды удаляют все данные безвозвратно. Сделать резервную копию перед выполнением.
# Сброс для Docker-варианта
docker compose -f deploy/docker-compose.prod.yml down -v
docker compose -f deploy/docker-compose.prod.yml up --build -d

# Сброс для нативного варианта (через psql)
psql -U postgres -c "DROP DATABASE feosport2;"
psql -U postgres -c "CREATE DATABASE feosport2 OWNER feosport;"
psql -U postgres -d feosport2 -f database\init.sql
psql -U postgres -d feosport2 -f database\seed.sql

9.4 Журналы установки

При проблемах с установкой проверить файл C:\Program Files\FeoSport2\install.log — он содержит подробный журнал всех операций.
